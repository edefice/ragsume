# From Front-End to AI Engineer: My Learning Path

## Overview

After a decade-plus in front-end and full-stack work, here's the deliberate path into AI
engineering — large language models, agents, and retrieval-augmented generation.

## Motivation

The decision came from practical necessity rather than industry panic: the tools I use every day
to do that work changed enough that I wanted to understand what's actually happening under the
hood. The goal wasn't to abandon web development, but to deepen the technical foundation by
understanding AI systems from the ground up.

## Learning Approach

Rather than attempting comprehensive machine learning mastery, I pursued three structured courses:

- Complete AI Engineer Training (Python, NLP basics, transformers, LangChain, Hugging Face)
- AI Engineer Agentic Track (agents and Model Context Protocol)
- AI Engineer Core Track (RAG, fine-tuning, agent design patterns)

Formal coursework was paired with daily practical application of Claude and Claude Code — shipping
working software instead of reading about it indefinitely.

## Transferable Skills

The web development background applies directly: API integration patterns, systems-level thinking
about data flow and failure modes, and product intuition about usability all carry over regardless
of the underlying technology.

## Current Status and Future Direction

No production experience yet with AI features in a live environment — this is a learner building a
foundation through coursework and personal projects, documented here with an emphasis on practical
understanding over theoretical abstraction.
