# Ask My CV

A small Next.js/TypeScript RAG (retrieval-augmented generation) app that answers questions about
Tomasz by retrieving relevant chunks from his CV and project write-ups, then generating a
grounded, structured answer with the Anthropic API. A self-referential portfolio piece: the AI
engineer demo that talks about the AI engineer.

## Architecture

```
User question (chat UI)
  → POST /api/chat
    → embed query (Voyage AI)
    → retrieve top-k chunks (cosine similarity over precomputed embeddings)
    → build prompt with retrieved context
    → call Claude with a forced tool-use call, matching the RagAnswer schema
    → return { answer, sources[], confidence }
  → UI renders the answer + cited source chunks
```

Ingestion (`scripts/ingest.ts`) is an **offline, build-time step** — it reads `content/`, chunks
each document by heading/paragraph with overlap, embeds every chunk with Voyage AI, and writes
`data/embeddings.json`. The running app just loads that file and does an in-memory cosine
similarity search — no external vector database, no runtime embedding of the corpus.

Structured output is enforced via **Anthropic tool use**: a single tool (`provide_answer`) with a
JSON-schema input matching `RagAnswer`, called with a forced `tool_choice`, then validated with Zod
before it reaches the client — rather than asking the model to emit JSON in prose.

## Stack

Next.js 16 (App Router), TypeScript, React 19, Tailwind CSS, `@anthropic-ai/sdk`, `voyageai`
(embeddings), Zod, Vitest.

## Project structure

```
/app
  /api/chat/route.ts       # RAG endpoint: embed → retrieve → generate → structured response
  /page.tsx                 # chat UI
  /components/ChatMessage.tsx
  /components/SourceCard.tsx
/lib
  /anthropic.ts              # Anthropic client + tool schema for structured output
  /embeddings.ts             # Voyage AI client wrapper
  /retriever.ts              # cosine similarity search over data/embeddings.json
  /chunk.ts                  # markdown chunking logic
/content
  cv.md
  /projects/*.md
/data
  embeddings.json             # generated by scripts/ingest.ts
/scripts
  ingest.ts                   # offline embedding pipeline
  eval.ts                      # retrieval quality eval
/tests
  chunk.test.ts
  retriever.test.ts
```

## Setup

Requires Node.js 20+.

```bash
npm install
cp .env.example .env.local   # fill in ANTHROPIC_API_KEY and VOYAGE_API_KEY
npm run ingest                # chunk + embed content/ → data/embeddings.json
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Commands

- `npm run dev` — local dev server
- `npm run ingest` — regenerate `data/embeddings.json` from `content/`
- `npm run eval` — check retrieval quality against a small hand-written Q&A set
- `npm run build` — production build
- `npm test` — run unit tests
- `npm run lint` — ESLint

## Environment variables

```
ANTHROPIC_API_KEY=   # https://console.anthropic.com/settings/keys
VOYAGE_API_KEY=      # https://dash.voyageai.com/api-keys
```

Never commit real keys — `.env.local` is gitignored; `.env.example` documents the required vars.

## Deploying

Deploy on [Vercel](https://vercel.com/new). Set `ANTHROPIC_API_KEY` and `VOYAGE_API_KEY` as
environment variables in the project settings, and commit `data/embeddings.json` (or run
`npm run ingest` as part of the build) so the deployed app has embeddings to search.
